Widget Tricks - ods-results

ods-results is a rather simple widget to list records from a dataset. They can be displayed as a list or cards.

ods-results widget documentation

Several Code Library resources rely on ods-results, for example the table module component, to list records in a table. The search module component, to display the results of a search query. The generators also use an ods-results internally to list dataset content.

Simple usage, list records !

ods-results returns an array: the list of records. Records are objects, and all the metas are accessible through the fields key.

ng-repeat directive loop over each record and “print” the HTML for each record. For more advanced usage, please refer to the AngularJs ng-repeat documentation here

This first simple resource lists districts in Brussels, Belgium.

HTML 
<div class="container">
    <ods-dataset-context context="belgiumdistricts, belgiumdistrictsinhab"
                         belgiumdistricts-dataset="belgium-statistical-districts"
                         belgiumdistricts-parameters="{'sort':'inh_km2','refine.commune':'BRUXELLES'}"
                         belgiumdistrictsinhab-dataset="belgium-statistical-districts"
                         belgiumdistrictsinhab-parameters="{'sort':'inhab','refine.commune':'BRUXELLES'}">

        <h3>10 Brussels districts</h3>
        <h4>The context used by ods-results has <code>sort: inh_km2</code> parameter (ie. sort by density)</h4>
        <div ods-results="records"
             ods-results-context="belgiumdistricts"
             ods-results-max="10">
            <ul>
                <li ng-repeat="record in records">
                    <strong>{{ record.fields.sector_fr }}</strong>, {{ record.fields.inhab | number }} inhabitants.
                    ({{ record.fields.inh_km2 | number }} inhab/km<sup>2</sup>)
                </li>
            </ul>
        </div>

        <h3>10 Brussels districts</h3>
        <h4>The context used by ods-results has a <code>sort: inhab</code> parameter (ie. sort by population)</h4>
        <div ods-results="records"
             ods-results-context="belgiumdistrictsinhab"
             ods-results-max="10">
            <ul>
                <li ng-repeat="record in records">
                    <strong>{{ record.fields.sector_fr }}</strong>, {{ record.fields.inhab | number }} inhabitants.
                    ({{ record.fields.inh_km2 | number }} inhab/km<sup>2</sup>)
                </li>
            </ul>
        </div>

    </ods-dataset-context>
</div>

Cards list

With ODS layout system, you can easily list results and print each record as a card.

Note that these codes also relies on this resource that over-rides this ODS layout.

It can then be combined with other widgets, by getting a metadata from the record, and feeding another widget with it. In this example, we get the shape of the record, and feed ods-geotooltip with its value.

HTML CSS 
<div class="container">
    <ods-dataset-context context="belgiumdistricts"
                         belgiumdistricts-dataset="belgium-statistical-districts"
                         belgiumdistricts-parameters="{'sort':'inhab','refine.commune':'BRUXELLES'}">

        <div class="row"
             ods-results="records"
             ods-results-context="belgiumdistricts"
             ods-results-max="10">
            <div class="col-md-4 col-sm-6"
                 ng-repeat="record in records">
                <div class="box white flat">
                    <h3>{{ record.fields.sector_fr }}</h3>
                    <p>
                        {{ record.fields.inhab | number }} inhabitants. ({{ record.fields.inh_km2 | number }} inhab/km<sup>2</sup>)
                    </p>
                    <ods-geotooltip geojson="record.fields.geo_shape">
                        See the district on a map <i class="fa fa-map-pin" aria-hidden="true"></i>
                    </ods-geotooltip>
                </div>
            </div>
        </div>

    </ods-dataset-context>
</div>

Pagination block

ods-pagination-block can be added to paginate your results.

Beware of setting the same number of results in your ods-results-max and per-page parameters.

HTML CSS 
<div class="container">
    <ods-dataset-context context="belgiumdistricts"
                         belgiumdistricts-dataset="belgium-statistical-districts"
                         belgiumdistricts-parameters="{'sort':'inhab','refine.commune':'BRUXELLES'}">

        <div class="row"
             ods-results="records"
             ods-results-context="belgiumdistricts"
             ods-results-max="12">
            <div class="col-md-4 col-sm-6"
                 ng-repeat="record in records">
                <div class="box white flat">
                    <h3>{{ record.fields.sector_fr }}</h3>
                    <p>
                        {{ record.fields.inhab | number }} inhabitants. ({{ record.fields.inh_km2 | number }} inhab/km<sup>2</sup>)
                    </p>
                    <ods-geotooltip geojson="record.fields.geo_shape">
                        See the district on a map <i class="fa fa-map-pin" aria-hidden="true"></i>
                    </ods-geotooltip>
                </div>
            </div>
        </div>
        <div class="row">
            <ods-pagination-block context="belgiumdistricts"
                                  per-page="12">
            </ods-pagination-block>
        </div>

    </ods-dataset-context>
</div>

More advanced cards with record images

This example get image from the dataset with ods-record-image. It also displays the cost if the meta is filled.

The geolocation at the bottom of the card displays the address if it exists, or the word “Location” if the field is empty.

HTML CSS 
<div class="container">
    <ods-dataset-context context="ctx"
                         ctx-dataset="principales-depenses-dequipements-issy"
                         ctx-parameters="{'q':'NOT #null(image)'}">

        <div class="row"
             ods-results="records"
             ods-results-context="ctx"
             ods-results-max="12">
            <div class="col-md-4 col-sm-6"
                 ng-repeat="record in records">
                <div class="box white flat">
                    <ods-record-image context="ctx"
                                      record="record"
                                      field="image"
                                      domain-url="https://userclub.opendatasoft.com"></ods-record-image>
                    <!-- domain-url is not necessary if your page is on the same domain than the dataset -->
                    <div class="box-content">
                        <div class="headers">
                            <h3>{{ record.fields.equipements }}</h3>
                            <p>
                                {{ record.fields.description }}
                            </p>
                        </div>
                        <dl class="row">
                            <dt class="col-xs-6 no-mb">Financial year</dt>
                            <dd class="col-xs-6 no-mb">{{ record.fields.exercices }}</dd>

                            <dt ng-if="record.fields.cout_pour_la_ville_en_euros"
                                class="col-xs-6 no-mb">Cost</dt>
                            <dd ng-if="record.fields.cout_pour_la_ville_en_euros"
                                class="col-xs-6 no-mb">{{ record.fields.cout_pour_la_ville_en_euros | number }} €</dd>

                            <dd class="col-xs-12 no-mb mt">
                                <ods-geotooltip record="record"><i class="fa fa-map-pin" aria-hidden="true"></i>
                                    {{ record.fields.adresse || "Location" }}</ods-geotooltip>
                            </dd>
                        </dl>
                    </div>
                </div>
            </div>
        </div>
        <div class="row">
            <ods-pagination-block context="ctx"
                                  per-page="12">
            </ods-pagination-block>
        </div>

    </ods-dataset-context>
</div>
